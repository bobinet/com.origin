##共同体模型
Food COOP

###概述
1. 一个自主成长的、分布式的食品网络。
2. 自主成长的节律：
	1. Boot：启动阶段，保底人到位后制定Lab阶段《工作计划及预算》，募集预算后进入Lab阶段。
	2. Lab：Lab运行至第一次品鉴样品发出，成立始发COOP进入COOP阶段。这个阶段主要是首发产品设计、始发成员招募。
	3. COOP：始发COOP成立至第二个COOP成立。这个阶段的工作重点开始转向建模和供应链管理。
	4. COOPs：多个COOP的完整模型。
	5. 共同体在进入COOPS阶段后召集后续阶段的定义，由Lab审议后编制发布。
3. 分布式的执行力：
	1. 由共同体根据外部事件生成工单；
	2. 职能角色承诺工单处理的时间和结果，刚性执行；
	3. 设立保底人角色，处理所有无人承诺的工单。

###角色清单
1. 召集人：负责本模型设计，发布《启动公告》。
2. 选题者：在《启动公告》发布一周内接受申请，召集人从中指定选题者。选题者有权：
	* 在Lab、COOP阶段任命Lab执行官、司库。
	* 批准Lab盘点
	* 根据Lab盘点结果提前结束工作周期。
3. 保底人
	* 共同体设置2名保底人：
		* 选题者自动成为第一保底人
		* 召集人自动成为第二保底人；
	* 保底人对共同体的承诺：
		1. 联手处理所有无人承诺的工单；
		2. 任何决议不能按时达成时，由保底人决策；
		3. 第二保底人的投入不低于第一保底人的150%；（折算为内部记账单位）
	* 召集人、选题者（及其兼任的保底人）是终身角色。
	* 如果召集人、选题者（及其兼任的保底人）不能履行职权，可以在一周内指定继任者并退位，否则由另一人在第二周指定继任者，如果两人均不指定，则由COOP管理员选举继任者。
3. 执行官：Lab、COOP阶段由选题者任命，COOPs阶段由各COOP管理员选举。负责
	1. 负责处理模型、规章为规定事务，签发通用工单。
	2. 编制财务报告。
4. 司库：Lab、COOP阶段由选题者任命，COOPs阶段由始发成员、各COOP管理员选举。负责
	1. 资金管理；
	2. 采购；
	3. 发货；
	4. 材料、产品、设备、设施、场地管理。
5. 产品研制者：由执行官任命，负责制订《产品规划》。
6. 品鉴者：由产品研发者、始发成员、各COOP管理员提名，执行官审批。负责
	1. 分发样品；
	2. 收集反馈，提交《品鉴反馈》。
7. 始发成员：在始发COOP之外第一个COOP成立前，提交申请加入始发COOP者。在退出共同体之前，始发成员除了普通成员的权利外，还享有司库选举权、品鉴者提名权，可以申请Lab盘点。
8. COOP
	1. 管理员：负责制定生产计划、管理COOP库存、采购、发货、及其它成员服务；
	2. 生产者：由管理员任命，负责生产；
	3. 监督员：负责监督COOP按模型和规章运行；
	4. 成员：缴纳年费并选举管理员、监督员。

###协作过程
* 启动
	1. 《启动公告》发布，共同体即进入Boot阶段；
	2. Boot第一周，召集第二保底人申请，由第一保底人从中指定第二保底人。
	3. Boot第二周，召集Lab阶段的《工作计划及预算》，保底人必须各自独立起草、提交；
	4. Boot第三周，保底人协商后共同发布Lab阶段的《工作计划及预算》正式稿；
	5. Boot第四周，启动预售募集Lab阶段预算：
		* 募集完成，共同体进入Lab阶段，宣布Lab成立并进入第一个工作周期；
		* 募集失败，退款并解散。

* Lab周期开始
	1. 成立后第一个周期的第一天完成执行官、司库、产品研制者任命，次日起开始正式工作；
	2. 正式工作：第一周完成**准备**，第二周开始进入**试制**。
* Lab准备：发出以下工单，通常一周内完成：
	* [执行官]接收品鉴者提名并完成审批，提交《审议意见》、《品鉴计划》；
	* [司库]准备设备设施场地，提交《入库单》；
	* [司库]采购食材，提交《入库单》。
* Lab试制：发出以下工单：
	* [产品研制者]产品试制，编制《产品规划》；
	* [执行官]审批样品及《产品规划》，重复直至通过。
* Lab品鉴
	* [司库]寄送样品及共同体介绍；（按品鉴者提供的名单，以及各COOP）
	* [品鉴者]收集反馈，提交《拓展报告》；
	* [执行官]根据反馈可以重启**准备**和**试制**环节的任务。
* Lab盘点
	1. [执行官、始发成员、COOP管理员、监督员]可以提出盘点申请，经选题者批准后可实施，由召集人指定代表点检各类资产，提交《审议意见》；
	2. [选题者]根据盘点结果可以提前结束工作周期，进入周期结束环节。
* Lab周期结束
	1. 完成一次盘点。
	2. [执行官、司库、产品研制者、品鉴者]提交《工作报告及决算》，执行官汇总； 
	3. [执行官、司库候选人]提交新工作周期的《工作计划及预算》；
	4. 按规定完成执行官、司库、产品研制者任命，次日起进入新的工作周期：
		* 执行官、司库：Lab、COOP阶段由选题者任命，COOPs阶段由始发成员、各COOP管理员选举。
		* 产品研制者：由执行官任命。

* COOP成立
	1. 始发COOP：
		1. Lab成立后第一个周期的第一次**Lab品鉴**样品发出后，由执行官成立始发COOP；
		2. 新成员提交《成员加入申请》并缴纳年费即成为成员，如果当时共同体处于COOP阶段，该成员为始发成员；
	2. COOP：
		1. 10名新成员联名提交《COOP成立申请》并缴纳年费，即可成立新的COOP；
		2. 始发COOP以外第一个COOP成立，共同体即进入COOPs阶段。
	3. 新COOP成立第一天完成团队构建。
	4. 新COOP成立第一周，管理员任命生产员，完成正式运行各项准备。
	4. 新COOP成立第二周起正式运行。
* COOP运行（重复以下步骤）
	1. [管理员]公布下周《产品清单》；
	2. [成员]提交《成员定单》；
	3. [管理员]根据《成员名录》、《成员定单》，制定《生产计划》、《采购计划》，修改《成员名录》中的专户余额；
	4. [生产者]完成产品生产；
	5. [管理员]组织发货并接受成员反馈。
* COOP盘点
	1. [COOP成员]可以提出盘点，监督员选择时间检各类资产，提交《审议意见》；
	2. [监督员]根据盘点结果可以启动COOP选举。
* COOP团队构建
	1. 出现以下情况时，COOP开始团队构建：
		1. 每年12月20日开始的常规换届；
		2. 监督员根据盘点结果启动改选；
		3. 管理员辞职；
		4. 超过三分之一COOP成员联名提出。
	2. 以下成员拥有选举权：
		* 联名提交《COOP成立申请》并缴纳年费的成员；
		* 该COOP《成员名录》中，团队构建开始前六个月加入的。成员
	3. 任何共同体成员拥有被选举权。
	4. 新COOP成立的团队构建在一天内完成，其它情况一周内完成，否则由执行官任命该COOP的管理员、监督者。

* 选举
	1. 选举主持人：
		* 执行官选举由选题者主持；
		* 其它选举由执行官主持。
	2. 参选者提交对应的《工作计划及预算》，由主持人确认其被选举权后公布；
	2. 拥有选举权的成员提交《审议意见》，明确选择的参选择；
	3. 主持人确认《审议意见》的有效性，公布汇总结果；
	4. 选举均按一人一票记票，得票超过拥有选举权的成员人数50%者胜出，如果所有参选择票数均不足，则由召集人发布《审议意见》直接任命。

* 制度设计
	1. 执行官每周根据《通用工单》、《通用日志》编写《审议意见》，明确本模型和具体规章、表单、代码实现的改进建议。
	2. 经召集人批准后，执行官可将上述改进方案写入《工作计划及预算》，正式实施。
